## 2. Data Model Design (Star Schema)

### Dimension Tables

#### DimDate (Date Dimension)

Create a calendar table for time intelligence:

```dax
DimDate =
ADDCOLUMNS(
    CALENDAR(DATE(2023, 1, 1), DATE(2025, 12, 31)),
    "Year", YEAR([Date]),
    "Quarter", "Q" & FORMAT([Date], "Q"),
    "Month", FORMAT([Date], "mmmm"),
    "MonthNumber", MONTH([Date]),
    "Week", WEEKNUM([Date]),
    "DayOfWeek", FORMAT([Date], "dddd"),
    "IsWeekend", IF(WEEKDAY([Date]) IN {1, 7}, TRUE, FALSE)
)
```

#### DimUsers (User Dimension)

- **Source**: `users` table
- **Key**: `user_id`
- **Attributes**: full_name, email, country, subscription_tier, signup_date, is_active

#### DimProducts (Product Category Dimension)

- **Source**: Extract from `transactions` table
- **Key**: `product_category`
- **Attributes**: category_name, category_group

#### DimTicketCategories

- **Source**: Extract from `support_tickets` table
- **Key**: `ticket_category`
- **Attributes**: category, priority_mapping, sla_targets

### Fact Tables

#### FactTransactions

- **Source**: `transactions` table
- **Grain**: One row per transaction
- **Keys**: transaction_id, user_id, transaction_date
- **Measures**: amount, transaction_status, payment_method, product_category

#### FactSupportTickets

- **Source**: `support_tickets` table
- **Grain**: One row per ticket
- **Keys**: ticket_id, user_id, created_date
- **Measures**: resolution_time_hours, satisfaction_rating, status

#### FactRefunds

- **Source**: `refunds` table
- **Grain**: One row per refund
- **Keys**: refund_id, transaction_id, user_id, refund_date
- **Measures**: refund_amount, processing_time_days, refund_status

### Relationships

```
DimDate[Date] → FactTransactions[transaction_date] (1:*)
DimDate[Date] → FactSupportTickets[created_date] (1:*)
DimDate[Date] → FactRefunds[refund_date] (1:*)

DimUsers[user_id] → FactTransactions[user_id] (1:*)
DimUsers[user_id] → FactSupportTickets[user_id] (1:*)
DimUsers[user_id] → FactRefunds[user_id] (1:*)

DimProducts[product_category] → FactTransactions[product_category] (1:*)
DimTicketCategories[ticket_category] → FactSupportTickets[ticket_category] (1:*)
```

## 3. DAX Measures for KPIs

### User Activity Metrics

```dax
// Daily Active Users
DAU =
DISTINCTCOUNT(FactTransactions[user_id])

// Monthly Active Users
MAU =
CALCULATE(
    DISTINCTCOUNT(FactTransactions[user_id]),
    DATESINPERIOD(DimDate[Date], LASTDATE(DimDate[Date]), -30, DAY)
)

// 7-Day Moving Average DAU
DAU_7DayMA =
AVERAGEX(
    DATESINPERIOD(DimDate[Date], LASTDATE(DimDate[Date]), -7, DAY),
    [DAU]
)
```

### Churn & Retention Metrics

```dax
// Churn Rate
ChurnRate =
VAR InactiveUsers =
    CALCULATE(
        COUNTROWS(DimUsers),
        DimUsers[is_active] = 0
    )
VAR TotalUsers = COUNTROWS(DimUsers)
RETURN
DIVIDE(InactiveUsers, TotalUsers, 0) * 100

// Retention Rate
RetentionRate =
VAR CohortSize = [Cohort Users]
VAR ActiveUsers = [Active Users in Period]
RETURN
DIVIDE(ActiveUsers, CohortSize, 0) * 100

// User Lifetime (Days)
AvgUserLifetime =
AVERAGEX(
    DimUsers,
    DATEDIFF(DimUsers[signup_date], TODAY(), DAY)
)
```

### Revenue & Transaction Metrics

```dax
// Total Revenue
TotalRevenue =
CALCULATE(
    SUM(FactTransactions[amount]),
    FactTransactions[transaction_status] = "success"
)

// Average Order Value
AvgOrderValue =
DIVIDE([TotalRevenue], [Transaction Count], 0)

// Transaction Success Rate
TransactionSuccessRate =
VAR SuccessCount =
    CALCULATE(
        COUNTROWS(FactTransactions),
        FactTransactions[transaction_status] = "success"
    )
VAR TotalTransactions = COUNTROWS(FactTransactions)
RETURN
DIVIDE(SuccessCount, TotalTransactions, 0) * 100

// Revenue Growth (MoM)
RevenueGrowthMoM =
VAR CurrentMonth = [TotalRevenue]
VAR PreviousMonth =
    CALCULATE(
        [TotalRevenue],
        DATEADD(DimDate[Date], -1, MONTH)
    )
RETURN
DIVIDE(CurrentMonth - PreviousMonth, PreviousMonth, 0) * 100
```

### Refund Metrics

```dax
// Refund Rate
RefundRate =
VAR RefundCount = COUNTROWS(FactRefunds)
VAR SuccessfulTransactions =
    CALCULATE(
        COUNTROWS(FactTransactions),
        FactTransactions[transaction_status] = "success"
    )
RETURN
DIVIDE(RefundCount, SuccessfulTransactions, 0) * 100

// Total Refund Amount
TotalRefunds =
CALCULATE(
    SUM(FactRefunds[refund_amount]),
    FactRefunds[refund_status] IN {"processed", "approved"}
)

// Refund Impact (% of Revenue)
RefundImpact =
DIVIDE([TotalRefunds], [TotalRevenue], 0) * 100

// Average Refund Processing Time
AvgRefundProcessingTime =
AVERAGE(FactRefunds[processing_time_days])
```

### Support Ticket Metrics

```dax
// Average Resolution Time (Hours)
AvgResolutionTime =
AVERAGE(FactSupportTickets[resolution_time_hours])

// SLA Compliance Rate
SLAComplianceRate =
VAR TicketsWithinSLA =
    CALCULATE(
        COUNTROWS(FactSupportTickets),
        FactSupportTickets[resolution_time_hours] <= 48  // Example: 48h SLA
    )
VAR TotalResolvedTickets =
    CALCULATE(
        COUNTROWS(FactSupportTickets),
        FactSupportTickets[status] = "resolved"
    )
RETURN
DIVIDE(TicketsWithinSLA, TotalResolvedTickets, 0) * 100

// Average Satisfaction Rating
AvgSatisfactionRating =
AVERAGE(FactSupportTickets[satisfaction_rating])

// Open Tickets
OpenTickets =
CALCULATE(
    COUNTROWS(FactSupportTickets),
    FactSupportTickets[status] IN {"open", "in_progress"}
)
```

### Customer Lifetime Value

```dax
// Customer Lifetime Value
CLV =
SUMX(
    DimUsers,
    CALCULATE(SUM(FactTransactions[amount]))
)

// Average CLV by Tier
AvgCLVByTier =
AVERAGEX(
    VALUES(DimUsers[subscription_tier]),
    [CLV]
)

// CLV per Active User
CLVPerActiveUser =
DIVIDE(
    [CLV],
    CALCULATE(COUNTROWS(DimUsers), DimUsers[is_active] = 1),
    0
)
```

## 6. Formatting & Styling

### Theme

Use a professional dark or corporate theme:

- Primary color: #366092 (Blue)
- Accent color: #E74C3C (Red for alerts)
- Success color: #27AE60 (Green)
- Background: Light gray (#F5F5F5)

### Conditional Formatting Rules

1. **Churn Rate**: Red if >15%, Yellow if 10-15%, Green if <10%
2. **SLA Compliance**: Red if <80%, Yellow if 80-95%, Green if >95%
3. **Transaction Success Rate**: Red if <90%, Yellow if 90-95%, Green if >95%
4. **Anomaly Severity**: Color-coded by severity level

### Data Labels

- Enable data labels on all bar/column charts
- Use percentages for rates, currency format for revenue
- Show only top N values for clarity (e.g., top 10 categories)

## 7. Performance Optimization

### Best Practices

1. **Use DirectQuery** for real-time data or **Import Mode** for faster performance
2. **Create aggregation tables** for large datasets (>1M rows)
3. **Remove unused columns** from data model
4. **Disable auto date/time** (use custom DimDate instead)
5. **Use measures** instead of calculated columns when possible

### Aggregations (for large datasets)

```dax
// Create aggregated table for daily metrics
DailyMetrics =
SUMMARIZE(
    FactTransactions,
    DimDate[Date],
    DimUsers[subscription_tier],
    "TotalRevenue", [TotalRevenue],
    "TransactionCount", [Transaction Count],
    "DAU", [DAU]
)
```

## 10. Example Use Cases

### Use Case 1: Identify Churn Risk

**Goal**: Find high-value users at risk of churning

**Steps**:

1. Navigate to **User Analytics** page
2. Apply slicer: subscription_tier = "premium" OR "enterprise"
3. Sort by days_since_last_login (descending)
4. Drill through to User Detail for top 10 users
5. Export list for retention campaign

### Use Case 2: Investigate Revenue Drop

**Goal**: Understand why revenue decreased last week

**Steps**:

1. Navigate to **Transaction Analytics** page
2. Adjust date slicer to last 14 days
3. Compare week-over-week revenue
4. Check Transaction Success Rate for anomalies
5. Review **Anomaly Dashboard** for spikes in failures
6. Drill down by payment method and product category

### Use Case 3: Support Team Performance

**Goal**: Evaluate support team efficiency

**Steps**:

1. Navigate to **Support Operations** page
2. Review Avg Resolution Time trend
3. Check SLA Compliance Rate gauge
4. Filter by ticket_category = "critical"
5. Identify bottlenecks (high resolution time categories)
6. Review satisfaction ratings correlation
